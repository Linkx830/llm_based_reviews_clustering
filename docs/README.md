# 文档目录说明

本文件夹包含项目的所有技术文档和规范文档。

## 文档列表

### 1. 系统架构设计文档.md

**作用**：详细描述系统的软件架构、设计模式和实现方法。

**内容**：
- 系统概述与目标
- 分层架构设计（应用层、Agent层、工具层、模型层、存储层）
- 核心组件设计（Orchestrator、IssueClusterAgent、ClusterInsightAgent等）
- 数据流与处理流程
- 关键技术实现（结构化输出、上下文处理、两阶段聚类等）
- 设计模式与原则
- 扩展性设计
- 性能与优化

**适用对象**：系统开发者、架构师、代码审查人员

---

### 2. 软件设计文档.md

**作用**：描述系统的业务设计、功能需求和实现方案。

**内容**：
- 项目目标与最终产出
- 数据字典（DuckDB表字段说明）
- 核心风险与改进策略
- 总体架构（LangChain + DuckDB）
- 数据流向时序图
- Agent划分与职责
- 中间表设计
- 执行流程
- 评估方案

**适用对象**：产品经理、业务分析师、项目管理者

---

### 3. 聚类规范.md

**作用**：详细说明聚类算法的实现规范和最佳实践。

**内容**：
- 两阶段聚类流程（Aspect分桶 + 桶内Issue聚类）
- 结构化输入设计
- Embedding生成与向量设计
- Reranker边界精修
- 簇后处理（噪点吸附、小簇合并）
- 参数调优建议

**适用对象**：算法工程师、数据科学家

---

### 4. 附录 A：LLM 结构化输出 Schema 规范（Extraction & Insight）.md

**作用**：定义LLM结构化输出的Schema规范。

**内容**：
- Extraction任务的输出Schema（AspectSentimentExtraction）
- Insight任务的输出Schema（ClusterInsightOutput）
- 字段说明与约束
- 验证规则

**适用对象**：LLM开发者、Prompt工程师

---

### 5. 附录 B：DuckDB 中间表与版本字段规范（复现、审计、断点续跑）.md

**作用**：定义DuckDB中间表的结构和版本字段规范。

**内容**：
- 11张中间表的完整Schema
- 版本字段说明（run_id, pipeline_version, data_slice_id）
- 表关系与数据流向
- 复现与审计机制

**适用对象**：数据库管理员、数据工程师

---

### 6. 项目文件规范.md

**作用**：说明项目文件的组织规范和命名约定。

**内容**：
- 目录结构说明
- 文件命名规范
- 代码组织原则
- README文件要求

**适用对象**：所有项目参与者

## 文档使用建议

1. **新加入项目的开发者**：
   - 先阅读`项目文件规范.md`了解项目结构
   - 阅读`软件设计文档.md`了解业务需求
   - 阅读`系统架构设计文档.md`了解技术实现

2. **算法工程师**：
   - 重点阅读`聚类规范.md`
   - 参考`附录 A`了解LLM输出格式

3. **数据工程师**：
   - 重点阅读`附录 B`了解数据库设计
   - 参考`系统架构设计文档.md`了解数据流

4. **代码审查人员**：
   - 参考`系统架构设计文档.md`中的设计模式与原则
   - 参考`项目文件规范.md`检查代码组织

---
